UAGithubEngine

  • Complete Objective-C wrapper around Github’s API.
  • Supports both Mac and iOS.
  • Makes full use of modern language features such as blocks and ARC.

NSManagedObjectContext+Utilities

  • NSManagedObjectContext category for working with JSON retrieved by a web service, provided the local data store is configured following a number of conventions:
    • The remote must use underscore case for its attributes; camel case must be used locally.
    • For remote attributes to be mirrored locally, the local attribute must use the same name as the remote attribute, adjusted for case. For example, my_object_attribute on the remote, and myObjectAttribute locally.
    • The value of remote attributes whose names would conflict with methods or properties on NSObject or NSManagedObject are stored locally with the prefix remoteObject. For example, id becomes remoteObjectID and description becomes remoteObjectDescription.

  • Allows developers to use the conventions of the relevant environments (e.g. Rails, iOS).

  • Fully documented in source files.

  • Example usage when paired with UAGithubEngine. The following code makes an API call and either creates or updates an NSManagedObject for the relevant entity from the returned data, adding it to an array for future use:

    NSMutable Array *managedObjects = [@[] mutableCopy];
[[UAGithubEngine newWithStoredCredentials] repository:[repository fullPath] success:^(id response){
    	[managedObjects addObject:[self.managedObjectContext managedObjectWithEntityName:@"IHRepository" fromDictionary:[response firstObject]]];
} failure:^(NSError *error){
    NSLog(@"%@", error);
}];

CBManagedObjectModelParser

  • Parses NSManagedObjectModel instances to provide an NSDictionary-based representation of a data structure.
  • Provides a method to create Rails models based on an NSManagedObjectModel instance.
  • Used to ensure uniformity between a local data structure and a web service.
  • Works well when used hand-in-hand with NSManagedObjectContext+Utilities.

UATableNavigationController

  • Collection of classes forming a Mac control with similar functionality to UINavigationController on iOS.
  • Intended to present a navigation stack similar to many iOS apps, useful for drill-down interfaces.
  • Provides methods to animate between table views, prompted by either single or double clicks.
  • Provides delegate methods for row single and double clicks.
